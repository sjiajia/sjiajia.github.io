这次博客的构建出现的问题如下：

##执行npm install hexo -g命令，一直报错，一直出现警告。

解决方法：我重新在另外一个盘中新建了一个文件夹，并在文该件夹下面打开了git.bash执行了以下命令：

$ hexo init
$ npm install

输入第一条命令时会出现：

Start Blogging with Hexo!

然后再输入第二条命令。发现输入该条命令后，虽然安装成功，但是仍然有报错：

npm WARN optional Skipping Failed optional dependence/chokidar/fsevents:
npm WARN notsup Not compatible with your operating system or architecture:

后来在群里的回复说这两个警告可以忽略，于是问题解决。

##执行命令hexo deploy时又出错了，而且看过群里面的相关回答却发现相关的解决办法都没用。

错误如下：

ERROR Deployer not found: github

解决办法：

1.type、repository、branch前面要加两个空格

2._config文件中type repository branch 三个冒号后面要加空格

3.type后的名字要用git

4.进行 npm install hexo-deployer-git –save 操作 后重新hexo d

我是进行了第四步操作才解决问题。

##经过之前的步骤后，我已经能够访问localhost：/4000网页，然而却是有的时候能够打开，有的时候却不能打开。

解决办法：

输了hexo server 之后 重新打开一个Bash 进去目录进行下面的操作，不要在进行hexo server操作时进行Ctrl c 操作，否则会打不开该网页。

##关于主题设置过程中的问题，我在导航栏上面新增加了两部分内容：
hexo new page “about”
hexo new page “feelings”
编辑了相关内容后，在主题的配置文件themes/cyansytle/_config.yml中添加：

menu：
  About： /about
  Feelings: /feelings

然而在博客的导航栏中点击相关选项，跳出来的却是出错界面404.

解决方法：

配置文件很注意字母的大小写，我将menu里的大写字母都换成了小写，并且设置了language，将其设置为英文，之后再点击相关选项，就能够进入了。

后来我又重新增加了两部分内容：

hexo new page “categories”
hexo new page “tag”

并且也在相应的配置文件中添加了

menu：
  categories： /categories
  tag: /tag
